💡 一句话定位：交换芯片是 AI 数据中心的「神经网络」—— Scale-out（Leaf/Spine）决定集群能连多少 GPU、跨柜带宽多大；Scale-up（NVSwitch）决定单个超节点内 GPU 协同效率。两层都依赖高端交换芯片，且共同拉动 光模块（Leaf↔Spine 光互联）需求。
🔁Scale-out vs Scale-up：两种交换芯片的本质区别
▼
AI 集群网络分两层：Scale-up 解决"柜内 GPU 怎么连"（超低延迟、超高带宽、内存语义）；Scale-out 解决"跨柜 GPU 怎么连"（高基数、可扩展、拓扑灵活）。两类交换芯片的设计目标、协议、玩家都完全不同。

💡 投资视角：
  • Scale-out 赛道：Broadcom 主导，催化看 102.4T 时间表 + Nvidia Spectrum-X 渗透速度 → 直接联动光模块（中际旭创/新易盛）。
  • Scale-up 赛道：Nvidia 凭 NVLink 独占 GPU 集群柜内市场，UALink 联盟是非 NV 阵营反击的关键 —— Broadcom Tomahawk-Ultra 进展、AMD MI 系列 + UALink 集群部署节奏是核心观察点。
  • 关键分叉：当 Scale-up 域规模扩大（如 NVL576），柜内距离突破 5 米 → 可能转向 CPO/LPO 光互联，那时 Scale-up 也会拉动光模块需求 —— 这是 2027+ 的远期看点。

🧬 什么是"协议生态"？为什么投资人必须看懂
协议生态 = 交换芯片"说哪种语言"。同样是搬数据，不同协议规定了帧格式、寻址方式、流控、重传、容错、内存语义 —— 协议决定了哪些芯片/设备能互联，也决定了谁是规则制定者。AI 算力时代有 6 大协议生态并存。
协议 阵营 主导/玩家 定位 状态
以太网 Ethernet 开放 IEEE 标准 / Broadcom 主导芯片 Scale-out 通用，AI/园区/企业全场景 主流
UEC（Ultra Ethernet） 开放联盟 AMD/Arista/Broadcom/Cisco/Meta/Microsoft 为 AI 优化的以太网，对抗 IB/NVLink 锁定 2025 标准落地
InfiniBand 半封闭 Nvidia（收购 Mellanox）独占 HPC + AI 训练首选 Scale-out 高端主流
NVLink / NVSwitch 私有锁定 Nvidia 完全私有 Scale-up 柜内 GPU 互联 独占
UALink 开放联盟 AMD/Broadcom/Cisco/Google/HPE/Intel/Meta/Microsoft 对标 NVLink 的 Scale-up 开放协议 1.0 已发，2026 量产
CXL / PCIe 开放 Intel/AMD/Arm + CXL 联盟 内存池化、CPU-加速器互联 配角，仅小范围 Scale-up
📌 协议生态会引发什么差异
① 客户议价能力
开放协议（以太/UEC/UALink）→ 多供应商竞争 → 云厂可砍价、可二供。
私有协议（NVLink/IB）→ Nvidia 锁定 → 高溢价、高粘性。
② 性能 vs 通用性
私有协议为特定硬件深度优化 → 延迟/带宽更极致。
开放协议为兼容性妥协 → 性能略低但生态广。
③ 站队效应
选 NVLink = 绑死 Nvidia GPU。
选 UALink = 拥抱 AMD/Intel + 多家芯片厂商。
云厂自研 GPU（Trainium/TPU）→ 倾向开放协议。
④ 谁定标准谁吃肉
Nvidia 凭 NVLink 收"协议税"。
Broadcom 主推 UEC/UALink 想"去 Nvidia 化"，把交换价值从 GPU 厂手里抢回来 —— 这是当前最大博弈。
💎 核心博弈：Nvidia（NVLink + IB 双护城河） vs 反 Nvidia 联盟（UEC + UALink）。前者代表客户：Microsoft、Oracle 大规模采买 GB200/GB300；后者代表客户：AWS、Meta、Google 自研 GPU + 拥抱开放协议。
跟踪信号：① UALink 1.0 量产芯片首发时间　② AWS/Meta 自研 GPU + UEC 集群部署比例　③ Microsoft 是否从纯 NV 阵营转向"NV+UALink"双轨。
📚 SerDes × 交换芯片：物理基础与代际跃迁
SerDes = Serializer/Deserializer（串行器/解串器），是位于交换芯片边缘的高速 I/O 模块，负责把芯片内部的并行数据转换成串行电信号送到光模块/铜缆 —— 它是交换芯片"对外吞吐"的物理通道。
📐 总容量公式
交换芯片总容量  =  单 SerDes 速率  ×  SerDes 数量
例：Tomahawk 5 → 100G × 512 lane ≈ 51.2T　｜　Tomahawk 6 → 200G × 512 lane ≈ 102.4T

① 物理基础
SerDes 速率翻倍 → 芯片总容量翻倍。没有 224G SerDes 就没有 102.4T 交换芯片。
② 光模块联动
交换芯片 SerDes 速率 = 光模块电口速率。224G SerDes 落地 → 1.6T 光模块（中际旭创/新易盛）放量。
③ IP 护城河
高速 SerDes IP 极少数厂商掌握：Synopsys / Cadence / Broadcom 自研 / Marvell（收购 Inphi）。是设计端壁垒。
💡 投资视角：研究交换芯片代际不能只看"总容量"，要看 SerDes 代际是否到位 —— 这是判断 102.4T 何时真正量产、1.6T 光模块何时放量的领先指标。例如 Broadcom 224G SerDes 客户出样进度，比 Tomahawk 6 流片新闻更早反映节奏。
🧱厂商核心能力拆解：SerDes 是关键吗？
▼
一句话：SerDes 是"必要不充分" —— 它是入场券和能力天花板，但不是护城河本身。交换芯片厂商真正的核心能力，是几个"难的东西"叠在一起的系统级整合。
✅ 为什么"必要"
到 200G/224G 这代，SerDes 决定你能不能做到 102.4T、信号能不能干净打出去。做不出领先 SerDes，连牌桌都上不了。
❌ 为什么"不充分"
Credo / Alphawave 有世界级 SerDes，却不是交换龙头——只有 I/O，没有交换架构和软件生态，最后只能卖 IP / 做 chiplet。
▍ 核心能力拆解（按护城河深浅排序）
能力是什么护城河谁强
① 软件 / 生态SDK、SONiC、网络 OS 集成、可编程(P4)、遥测最深换芯片要重写一堆软件，客户极不愿动 → 博通碾压
⑤ 客户绑定design win 多年锁定，进了集群难替换深在位者通吃
② 交换架构buffer 架构、调度、拥塞控制、自适应路由、无损网络深AI fabric 丢包/负载均衡硬功夫 → 博通(TH+Jericho)、中兴微(OLink)
③ SerDes高速 I/O、信号完整性中是天花板，但可买 IP 追赶 → 博通、Marvell、Nvidia
④ 制程+封装+规模拿 TSMC 3nm/CoWoS 产能、每代 R&D 烧几十亿中靠体量 → 博通、Nvidia
💡 真正的胜负手是 ①软件生态 + ⑤客户绑定 —— 这才是博通守住 ~70% 份额的根本。SerDes 强只是让你"有资格被考虑"。
▍ 核心能力因商业模式而异（别一把尺子量所有人）
厂商 / 模式核心能力命脉
博通 / Marvell

Nvidia

思科

🎯 三层投资判断框架
看一家交换芯片厂商，别只盯 SerDes 速率（那只是性能参数），要问三层：
① 技术够不够格（SerDes + 架构）—— 入场券
② 软件生态 / 客户有没有绑死 —— 真护城河
③ 商业模式决定它靠什么赚钱（单卖芯片 / 搭 GPU / 搭系统 / 国产替代）

▍ 三个量价关键点
① 拼接反而利好国产
博通 51.2T 缺货 → 2 颗 25.6T 替 1 颗 51.2T → 芯片用量翻倍 → 盛科 25.6T 独家直接受益（阿里 3 万颗/9 亿订单已落地）。
② 2027 演进再上台阶
256 卡（阿里）/ 144-168 卡（字节）/ 72-144 卡（腾讯）→ 单超节点 GPU 翻倍 → 交换芯片需求再上一个台阶。
③ 万卡 → 10 万卡
三大厂已启动万卡集群，后续按大模型需求向 10 万卡级演进，超节点是核心需求方，交换芯片量价齐升。
📎 假设与口径：① 国产 GPU 全年 400 万颗为假设值；② 超节点台数为厂商 2026 规划，颗数按"交换节点数 × 台数"推算（~8 万颗与公开 25.6T 需求吻合）；③ 非超节点 1:30 配比为估算（scale-out 稀疏区间 1:20~50，是最大不确定项，实际随训练/推理结构变化）；④ 价值量按 25.6T=3 万元/颗（盛科阿里订单单价）测算，非超节点若用低端芯片实际更低；⑤ 超节点 GPU:交换芯片（5-8:1）高于 NVL72 的 4:1，因国产用 400G 以太组网，单卡网络含硅量低于 NVLink。
